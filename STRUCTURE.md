# Repository Structure

Each week follows this structure:

```
week-N/
├── PROJECT_NAME.py          # Main mini-project for the week
├── week_N_exercises.py      # Exercises for practice
├── WEEK_N_NOTES.md          # Theory summary and key concepts
└── utils/                   # (If needed) Helper modules
```

## Weeks Overview

### Month 1: Fundamentals

- **Week 1:** Language Basics → Unit Converter
- **Week 2:** Conditions & Loops → Mystery Number Game
- **Week 3:** Functions → Scientific Calculator
- **Week 4:** Strings, Lists & Files → Task Manager (To-Do List)

### Month 2: Data Structures & OOP

- **Week 5:** Advanced Data Structures → Word Frequency Analyzer
- **Week 6:** Advanced Functions & Modules → Utility Functions Library
- **Week 7:** OOP Basics → Library Management System
- **Week 8:** Error Handling & Data Formats → Contact Manager

### Month 3: Professional Tools & Capstone

- **Week 9:** Professional Environment → Set up Git, GitHub, venv
- **Week 10:** APIs & Web Requests → Weather App
- **Week 11:** Testing & Code Quality → Test previous projects
- **Week 12:** Capstone Project → Your choice!

## How to Use This Repo

1. **Follow the week order** — start with Week 1
2. **Read WEEK_N_NOTES.md first** to understand concepts (~20 min)
3. **Do week_N_exercises.py** to practice (~1-2 hours)
4. **Build PROJECT_NAME.py** — the main deliverable (~3-5 hours)
5. **Commit and push** to GitHub when done
6. **Move to next week**

## General Workflow

```bash
# Clone this repo (first time only)
git clone https://github.com/yanwrld08/python-apprentissage.git
cd python-apprentissage

# Each week:
cd week-N
python week_N_exercises.py  # Test the exercises
python PROJECT_NAME.py      # Run the mini-project

# When satisfied, commit and push
git add week-N/
git commit -m "Week N: Complete [PROJECT_NAME]"
git push origin main
```

## Pro Tips

- Comment your code! Explain *why*, not just *what*
- Test with different inputs
- If something breaks, read the error message carefully
- Google the error message + "python" — 90% of errors are documented
- Don't move forward until you understand the previous week
- Revisit old projects occasionally and improve them
